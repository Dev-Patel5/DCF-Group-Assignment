Internet Group Management Protocol (IGMP)

What is IGMP?


The Internet Group Management Protocol (IGMP) is a communications protocol used by hosts and adjacent routers on IPv4 networks to establish multicast group memberships. A protocol which is for sharing an IP address among multiple devices so they can receive the same data. Internet Protocol version 4 (IPv4) networks use IGMP to configure multicasting at the network layer. The IGMP protocol enables devices to join multicast groups.


How does IGMP work?


IGMP is an integral part of IP multicast and allows the network to direct multicast transmissions only to hosts that have requested them. IGMP can be used for one-to-many networking applications such as online streaming video and gaming, and allows more efficient use of resources when supporting these types of applications.

Multicasting


Multicasting: Defination, Uses & Examples


The term multicasting refers to the sharing of messages or packets among a group of devices. IP addresses are shared between devices using multicasting.In networking, multicasting refers to the process of sending a single message or packet to multiple recipients at the same time. Multicasting is useful in situations where you want to send the same message or data to multiple recipients simultaneously, but you don't want to send separate copies of the message to each recipient. This can help to save bandwidth and improve the efficiency of the network. For example, multicasting can be used in video streaming applications to send the same video stream to multiple viewers at the same time, without having to send separate copies of the stream to each viewer. It can also be used in online gaming to send data to multiple players in a game simultaneously, or in other real-time communication applications. To use multicasting, the sender and recipients must be part of the same multicast group. Below are some other castings that contrast with multicasting.


BroadCast, Anycast and Unicast


Broadcast: Networked communication with a single sender and several recipients is said to be the broadcast addressing.

Anycast: Messages are routed to a single device within a certain group of devices using anycast addressing.

Unicast: The unicast addressing system states that network communication only involves one source and one recipient.

Relation to IGMP


In order to provide effective multicast routing on a network, IGMP and snooping cooperate. Snooping is a technique used by the network's switches and routers to track IGMP traffic and determine which hosts have joined which multicast groups. Instead of broadcasting the multicast packets to every device on the network, this information is then utilised to forward the packets just to the receivers who have asked for them.

Networking


Functionality


Networked devices such as computers and routers use IGMP when joining multicast groups. A router that implements IGMP can identify the devices in a multicast group by listening to their IGMP messages. In IGMP, IP addresses designated for multicasting are used. Multiplecast addresses range from 224.0.0.0 to 239.255.255.255. There are two IP addresses shared by each multicast group (anycast networks can use any IP address). The router will make copies of each packet it receives that is destined for the shared IP address and send them to each member of the multicast group. It is possible to modify IGMP multicast groups dynamically. A gadget may issue an IGMP "join group" request or at any time a "leave group" message. Directly built upon Internet Protocol, IGMP (IP). Both an IGMP header and an IP header are present in every IGMP packet. IGMP does not utilise a transport layer protocol like TCP or UDP, much like ICMP does not.

Internet Protocol (IP)


A set of guidelines called the Internet Protocol (IP) assists in rerouting data packets across networks so they may get to where they're supposed to. A computer divides the information into smaller units, known as packets, before delivering it. Each packet contains the IP address to make sure it is delivered to the right place.


IPV4 and IPV6


The two versions of the Internet Protocol (IP), the main protocol for delivering data over the internet, are IPv4 and IPv6. The fourth iteration of the original 1981-define IP protocol is known as IPv4. It can support up to 4.3 billion different addresses and uses 32-bit addresses. The most used IP protocol is IPv4, which is supported by the majority of networking hardware and internet service providers. Contrarily, IPv6 is the most current and sixth iteration of the 1998-define IP protocol. It makes use of 128-bit addresses and supports almost an infinite number of different addresses. IPv6 is gradually replacing IPv4 in order to address the rising issue of IPv4 address depletion.IPv6 is being gradually embraced by internet service providers and networking equipment suppliers in order to solve the rising issue of IPv4 address depletion. Despite the fact that IPv4 and IPv6 are both used to transmit data over the internet, IPv6 is regarded as IPv4's replacement and is likely to happen sooner rather than later. This is because IPv6 has a significantly bigger address space and offers more features and advancements than IPv4, such as support for autoconfiguration and enhanced security. Although it may take some time to complete, the switch from IPv4 to IPv6 will be gradual.
